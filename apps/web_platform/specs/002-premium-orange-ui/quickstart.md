# Quickstart: 平台高级化与表单视觉统一

**Feature**: `002-premium-orange-ui` | **Date**: 2025-03-27

## 前置

- 本地启动 `apps/web_platform`，已登录。
- 具备创建项目权限与至少一个项目（用于测「新建页面」）。

## 检查清单

### A. 创建项目 Modal

1. 打开 **项目管理** → **新建项目**。
2. 对照 [platform-form-shell.md](./contracts/platform-form-shell.md) 表格 **1～8** 项，与 **新建页面** Modal 逐项对比是否一致。
3. 故意留空必填项提交，确认错误提示位置可读。
4. 切换语言（若应用支持），确认 Modal 内文案有英文（或产品约定语言）。

### B. 新建页面 Modal

1. 进入 **项目详情** → **页面管理** → **新建页面**。
2. 重复 A 中与创建项目的 **视觉一致性** 对比。
3. 成功创建一页，确认列表刷新与 Toast 正常。

### C. 主界面（P3）

1. 浏览 **项目列表**、**项目详情**：顶栏、主内容背景、Card/Table 是否与暖橙协调、链接非冷紫主导。
2. 窗口宽度约 **1366px**：检查标题区、表格是否挤压或错位。

### D. 回归

- 创建项目、创建页面 **取消** 与 **成功提交** 均无控制台报错（与本次样式相关）。

## 完成定义

- `platform-form-shell.md` 中创建类 Modal **8** 项在走查表上标记为通过或已登记例外。
- spec **SC-004**：核心路径无新增阻塞（手工确认）。
