## Clarifications

### Session 2025-03-27

- Q: 改版范围是否包含注册/重置密码等认证子页（与登录页是否同版式、同动画）？ → A: Option C — 仅登录页采用完整新版（左右分栏 + 橙子角色视线动画）；注册、重置密码等其它认证相关页仅统一配色与字体等全局观感，不采用左右分栏布局，不实现橙子视线追踪动画。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 在新版登录页完成登录 (Priority: P1)

未登录用户进入登录页，在清晰、简洁的表单区域输入凭据并完成登录，进入与当前产品一致的后登录体验。

**Why this priority**: 登录是门禁能力；视觉升级不得削弱或破坏成功登录路径。

**Independent Test**: 仅启用新版布局与样式、保持现有登录接口与校验规则时，使用有效/无效凭据各执行一次完整登录尝试，可独立验证成功跳转与错误提示仍符合预期。

**Acceptance Scenarios**:

1. **Given** 用户未登录且打开登录页，**When** 在右侧（宽屏下的表单区）输入正确凭据并提交，**Then** 用户进入与改版前一致的已登录后目标流程。
2. **Given** 用户未登录且打开登录页，**When** 输入错误凭据并提交，**Then** 用户看到明确错误反馈且仍停留在登录页，可修正后重试。

---

### User Story 2 - 了解 OrangeHome 低代码价值 (Priority: P2)

用户在登录前浏览左侧面板，快速理解 OrangeHome 作为低代码系统能解决什么问题、适合谁使用。

**Why this priority**: 提升品牌感知与信任，支撑“高级”首屏体验，但不阻断登录主路径。

**Independent Test**: 在不提交表单的情况下，仅阅读左栏文案与视觉层次，由评审对照 FR-003 核对要点数量与可读性即可独立验收。

**Acceptance Scenarios**:

1. **Given** 用户在宽屏下打开登录页，**When** 视线从左向右浏览，**Then** 能先看到产品介绍要点，再自然注意到右侧登录区。
2. **Given** 用户仅关心登录，**When** 直接进入右侧表单操作，**Then** 不被无关信息阻碍完成登录（表单区保持简洁）。

---

### User Story 3 - 与橙子品牌角色互动 (Priority: P3)

用户在左栏看到橙子形象的品牌角色；在使用指针设备时，角色的眼部或视线在视觉上跟随指针位置，增强趣味与记忆点。

**Why this priority**: 强化品牌差异化与情感连接，属于增值体验，在资源或约束下可次于 P1/P2。

**Independent Test**: 在左栏可见、指针设备可用时，移动光标跨越面板主要区域，通过观察或短视频记录即可验证视线跟随是否可感知；在开启系统“减少动态效果”时验证替代呈现（见 Edge Cases）。

**Acceptance Scenarios**:

1. **Given** 用户使用指针设备且左栏可见，**When** 在左栏区域内移动光标，**Then** 橙子角色的眼部朝向变化可被肉眼察觉地与指针位置关联。
2. **Given** 用户启用系统“减少动态效果”（或等价无障碍设置），**When** 打开登录页，**Then** 页面信息完整可读，且不依赖持续的视线追踪动画才能理解品牌区内容。

### Edge Cases

- **触摸或无可移动光标**：无精确指针时，眼部追踪效果可不表现或改为轻微入场动画；不得因此遮挡表单或产品介绍正文。
- **窄视口 / 小屏**：左右分栏改为纵向堆叠（例如先品牌与介绍、后表单），关键信息与可点击目标仍满足可读性与可点区域尺寸。
- **高对比与可读性**：在温和橙色调下，正文、标签、占位符与错误提示仍满足团队约定的可读性底线（可与设计走查表对照）。
- **已登录用户访问登录 URL**：行为与当前产品一致（例如重定向到已登录首页），不因新版视觉而改变路由策略。
- **注册/重置密码等认证子页**：在配色、字体与品牌气质上与登录页协调，但页面结构保持现有形态；不得出现与登录页相同的左右分栏营销区或橙子视线追踪，以免验收范围蔓延。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 在典型桌面宽屏视口下，**登录页** MUST 呈现左右分区：左侧为 OrangeHome 产品介绍与品牌视觉（含橙子角色），右侧为登录表单区域。
- **FR-002**: **登录页**整页视觉 MUST 以温和、偏暖的橙色为主色调，且正文、标签与关键控件与背景对比度足以保证可读性（须通过设计/无障碍走查确认）。
- **FR-003**: 左侧产品介绍 MUST 以简明要点形式呈现 OrangeHome 低代码系统的核心能力或价值主张，要点数量为 3～6 条，文案可由产品后续微调但结构保留。
- **FR-004**: 左侧 MUST 展示可识别的橙子品牌角色；在指针设备可用且用户未声明减少动态效果时，角色的眼部或视线 MUST 在视觉上可感知地随指针位置变化；在用户声明减少动态效果时，MUST 提供不依赖持续追踪的静态或极简替代呈现，且信息完整。
- **FR-005**: 右侧登录区 MUST 高级且简洁：仅保留完成登录所需的字段、主操作与必要的辅助链接（如注册、找回密码，若当前产品已提供），避免营销堆砌或与登录无关的模块。
- **FR-006**: 登录成功与失败的行为 MUST 与改版前一致（校验规则、错误语义、跳转目标），除非另有独立变更规格。
- **FR-007**: 本页新增或变更的面向用户文案 MUST 同时提供中文与英文版本，与产品现有语言策略一致（参见项目宪章中的 i18n 要求）。
- **FR-008**: 注册、重置密码等**非登录页**的认证相关页面 MUST 在配色、字体与整体品牌气质上与登录页新版对齐；MUST NOT 引入与登录页相同的左右分栏产品介绍区；MUST NOT 实现橙子角色或等效的视线追踪动画。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在典型桌面视口下，未提前说明布局的抽样用户（内部 3～5 人即可）中，至少 80% 在 5 秒内能正确指出“哪一侧是登录、哪一侧是产品介绍”。
- **SC-002**: 在指针设备上，对左栏品牌区进行一次有目的的走查（光标跨越主要区域），评审一致认定橙子角色眼部朝向变化与指针位置存在可感知关联。
- **SC-003**: 抽样用户中至少 90% 能在无额外口头提示的情况下，使用有效凭据完成一次成功登录（可与 SC-001 同批走查）。
- **SC-004**: 在系统开启“减少动态效果”时，登录页核心任务（阅读介绍、完成登录）仍可完成，且不因缺少视线追踪而导致信息缺失或表单不可用。

## Assumptions

- 改版范围：**登录页**为完整新版（布局、角色、动画、文案）；**注册/重置密码等页**仅做全局视觉对齐（配色、字体等），结构沿用现状；不更改后端认证协议或业务规则。
- 目标用户具备常见桌面浏览器环境；移动端以可用降级为主，不要求与桌面完全相同的分栏比例。
- 产品介绍要点文案可由产品/运营提供初稿，交付前完成中英对齐。
- 橙子角色与动画为前端呈现资产，不涉及新增用户个人数据的采集或存储。
