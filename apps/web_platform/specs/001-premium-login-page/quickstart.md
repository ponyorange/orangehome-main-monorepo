# Quickstart: 登录页品牌升级验收

**Feature**: `001-premium-login-page` | **Date**: 2025-03-27

## 前置条件

- 本地启动 `apps/web_platform`（如 `pnpm` / Rush 约定命令 + `vite`，端口以 `vite.config.ts` 为准）。
- 可选：准备有效测试账号，用于成功登录路径。

## 手工检查清单

1. **宽屏分栏（≥1024px）**  
   - 打开 `/login`。  
   - 确认左侧为介绍 + 橙子角色，右侧为表单；整体为温和暖橙调，正文与标签可读。

2. **登录主路径（P1）**  
   - 输入错误凭据 → 明确错误提示，留在本页。  
   - 输入正确凭据 → Toast/反馈与现网一致，跳转 **`/projects`**（或当前实现等价目标）。

3. **产品介绍（P2）**  
   - 左栏可见 **3～6** 条要点；切换语言（若应用支持）时中英文均有对应文案。

4. **橙子视线（P3）**  
   - 在左栏内移动鼠标 → 眼部朝向变化可感知。  
   - 将鼠标移入右栏 → 行为符合实现约定（例如停止追踪或仅左栏监听），且不影响输入。

5. **减少动态效果**  
   - 在系统设置中开启「减少动态效果」/ Windows 相应选项，刷新 `/login`。  
   - 确认无依赖持续视线追踪才能读懂的介绍；表单可用。

6. **窄屏**  
   - 将视口缩至 &lt;768px（或按计划断点）。  
   - 确认纵向堆叠：先品牌区再表单，无横向溢出；主按钮可点。

7. **触摸 / 粗指针（可选）**  
   - 使用触摸或开发者工具模拟 → 无卡顿或错误假设「必有鼠标」的交互。

8. **注册 / 重置密码页（FR-008）**  
   - 打开 `/register`、`/reset-password`。  
   - 确认 **无** 左右分栏营销区、**无** 橙子视线动画；配色/字体与登录页 **协调**。

9. **已登录访问 `/login`**  
   - 行为与改版前一致（如重定向），不因新 UI 失效。

## 完成定义

- 以上与 `spec.md` 成功标准不冲突；无控制台报错（与本次改动相关）。
