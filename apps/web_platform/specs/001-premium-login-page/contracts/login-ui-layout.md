# UI Contract: 登录页布局与行为

**Feature**: `001-premium-login-page`  
**Applies to**: `src/pages/Login`（及其实际引用的子组件）  
**Date**: 2025-03-27

## 视口与结构

| 条件 | 布局 |
|------|------|
| width ≥ 1024px | **两列**：左「品牌 + 介绍 + 橙子角色」；右「登录表单 + 辅助链接」 |
| width < 768px（及可选 768–1023） | **单列纵向**：先左栏内容块（含静态或简化角色），再表单 |

## 左栏（登录页专属）

- 包含 **3～6** 条产品介绍要点（文案来自 i18n）。
- 包含 **橙子品牌角色**；在 `(pointer: fine)` 且非 `prefers-reduced-motion: reduce` 时，**眼部/视线**随指针在左栏区域内移动而可感知变化。
- 左栏内容 **不得** 遮挡右栏表单（宽屏下分列；窄屏下顺序展示）。

## 右栏

- 仅含登录所需字段、主提交操作、以及指向注册/重置密码的链接（若产品已提供对应路由）。
- 视觉 **简洁**，无与登录无关的营销模块。

## 认证子页（Register / ResetPassword）

- **不得** 使用本契约的「两列左栏」结构。
- **不得** 实现橙子视线追踪或等效交互。
- **应当** 通过共享主题 token（如 `auth-theme.scss`）与登录页在 **主色、背景气质、字体层级** 上协调。

## 无障碍与系统设置

- `prefers-reduced-motion: reduce`：**禁止**依赖持续指针追踪作为唯一品牌表达；信息完整可读。
- 表单控件、链接须保持键盘可操作性与可见焦点（Semi 默认行为需保留，勿用 `outline: none` 无替代）。
