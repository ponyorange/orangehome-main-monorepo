# Data Model: 登录页品牌与布局升级

**Feature**: `001-premium-login-page` | **Date**: 2025-03-27

## 持久化实体

本特性 **不引入** 新的服务端实体、数据库表或 API 载荷变更。登录仍使用现有凭据字段（如邮箱、密码）与既有认证接口。

## 前端展示数据（非持久化）

| 概念 | 说明 | 来源 |
|------|------|------|
| 产品介绍要点 | 3～6 条可本地化字符串 | `i18n` 资源，由产品维护文案 |
| 吉祥物姿态 | 指针相对位置 → 眼部 transform | 客户端计算，不落库 |

## 状态与生命周期

- **认证状态**：继续使用 `authStore`；成功登录后跳转 **`/projects`**（与当前 `pages/Login` 行为一致）。
- **表单状态**：Semi `Form` 本地状态；加载中由现有 `loading` 布尔控制。

无需单独 ER 图或迁移脚本。
