# Feature Specification: HTTP 环境下分享预览链接与复制失败兜底

**Feature Branch**: `011-http-share-fallback`  
**Created**: 2026-03-29  
**Status**: Draft  
**Input**: User description: "点击header的「分享」按钮时，提示复制失败，请检查浏览器权限或使用HTTPS。解决一下，我目前是用http协议，我需要可以复制链接。同时，如果还是复制失败，弹窗展示链接让用户手动复制。同时，弹窗有个跳转按钮，可以打开新的tab打开这个url。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 在 HTTP 访问下仍能完成「分享预览链接」(Priority: P1)

作为在本地或非 HTTPS 环境调试的搭建者，我点击页头（或同等主区域）的「分享」后，应能在**不依赖仅适用于安全访问环境的单一能力**的前提下，拿到**与当前页面上下文对应的可分享预览链接**（在链接本身可生成的条件下），而不是只看到要求我改用 HTTPS 的终止性提示。

**Why this priority**：当前痛点直接阻断分享流程，影响日常开发与内网演示。

**Independent Test**：在仅使用 `http://` 访问应用时点击「分享」；若预览链接可生成，用户应能获得可复制的链接或等价可交付形式，且流程可继续，而非仅失败提示结束。

**Acceptance Scenarios**:

1. **Given** 用户通过 HTTP 访问且系统能根据当前上下文生成预览链接，**When** 用户点击「分享」，**Then** 系统应尝试将链接交付给用户（例如通过系统允许的复制方式），且**不得**将「必须使用 HTTPS」作为**唯一**结果而终止流程（当仍存在其他合理交付方式时）。
2. **Given** 用户通过 HTTPS 访问且链接可生成，**When** 用户点击「分享」，**Then** 行为应与改进前「成功分享」的用户预期一致或更优（成功反馈、链接正确）。

---

### User Story 2 - 自动复制失败时用弹窗展示链接并可新开页打开 (Priority: P1)

作为搭建者，当自动复制仍因环境或权限失败时，我希望弹出对话框**完整展示**该预览链接，便于我手动选中复制；同时我希望有一键在**新浏览标签页**中打开该链接，以便快速验证。

**Why this priority**：与 US1 共同构成完整闭环；手动复制与打开链接是常见兜底，减少支持成本。

**Independent Test**：模拟或触发「无法完成自动复制」的情形后点击「分享」；应出现包含完整 URL 的弹窗，用户可阅读/手动复制，且「在新标签页打开」能打开同一 URL。

**Acceptance Scenarios**:

1. **Given** 预览链接已生成且自动复制未成功，**When** 系统进入兜底路径，**Then** 用户应在弹窗中看到**完整、可读**的链接文本（或可等价获取完整字符串的方式）。
2. **Given** 弹窗已展示链接，**When** 用户选择「在新标签页打开」类操作，**Then** 系统应在新的浏览标签页中导航至该链接所指向的地址。
3. **Given** 弹窗已展示链接，**When** 用户关闭弹窗，**Then** 不应残留阻塞编辑的界面状态（除产品另有全局规则外）。

---

### Edge Cases

- **无法生成预览链接**（缺少页面标识、未配置模板等）：应保持与现有产品一致的明确说明，本规格不强制改变「无 URL」时的文案，但不得与 US1/US2 的成功路径混淆。
- **链接极长**：弹窗内链接应可滚动或换行展示，避免不可读。
- **用户连续多次点击「分享」**：行为应可预期（不叠加多个无法关闭的弹窗，或按产品统一模式处理）。
- **浏览器禁止弹窗**：「新标签页打开」若被拦截，应有可理解的失败反馈或引导（与现有产品错误处理风格一致即可）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**：当预览分享链接**能够**根据当前上下文生成时，系统 MUST 在用户点击「分享」后**交付**该链接；在仅 HTTP 访问的场景下，MUST 采用**不止一种**可行策略中的至少一种，使链接可被用户获得（含自动复制成功或进入兜底弹窗等），而不得仅依赖「要求改用 HTTPS」作为唯一出口。
- **FR-002**：当**自动复制**未能完成时，系统 MUST 以**弹窗（或同等模态）**向用户展示**完整 URL**，以便手动复制。
- **FR-003**：在上述兜底弹窗中，系统 MUST 提供**在新标签页打开该 URL** 的明确操作入口。
- **FR-004**：当自动复制**成功**时，系统 SHOULD 继续提供清晰的成功反馈（可与现有文案一致），且不应**无故**再弹出完整 URL 弹窗打断用户（除非产品明确选择「始终二次确认」模式——默认不采用）。
- **FR-005**：所有失败类提示 MUST **可理解**且与实际情况一致（例如区分「无链接可生成」「复制失败」「打开新标签被拦截」），避免误导用户认为仅能通过 HTTPS 解决而实际已有兜底路径。

### Key Entities

- **预览分享链接**：面向运行时预览的完整 URL 字符串；由当前编辑上下文与产品配置的链接规则决定是否存在、内容为何。

## Orange Editor / web_builder *(conditional)*

- **ARCH-001**：实现须遵循 `.specify/memory/constitution.md`（扩展边界、数据与 UI 分层等）。
- **ARCH-002**：本能力作用于编辑器内提供「分享」预览链接的**主入口**（与现有「分享」按钮一致）；若存在多个入口，默认**行为一致**，除非后续规格单列例外。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**：在**仅 HTTP** 访问、且预览链接可生成的走查中，**至少 90%** 的试用次用户能在**两次以内**的「分享」操作内获得可复制或可打开的链接（自动复制成功或经弹窗手动复制/打开）。
- **SC-002**：在刻意触发「自动复制失败」的场景下，**100%** 走查中用户能在弹窗中看到与预期一致的完整 URL，且「新标签页打开」在未被浏览器拦截时**一次成功**。
- **SC-003**：走查中**零例**出现「仅提示必须使用 HTTPS、且无任何链接展示或其它交付方式」而用户实际处于可生成链接的上下文。

## Assumptions

- 「分享」所指为当前产品中用于**运行时预览链接**的分享能力，与导出 JSON 等其它分享形式无关。
- 预览链接是否可生成仍受现有业务规则约束（如页面 ID、环境变量/配置模板）；本规格只约束**可生成时**的交付与兜底体验。
- 主要使用场景为桌面浏览器；移动端浏览器的能力差异可通过与桌面一致的兜底弹窗覆盖，不单独开子项目。
- 用户理解「手动复制」需要自行使用系统或浏览器的选区与复制操作（规格不要求内置二次「点按钮再复制」除非作为增强）。
