# Feature Specification: 画布区组件悬停虚线框

**Feature Branch**: `009-canvas-hover-outline`  
**Created**: 2026-03-29  
**Status**: Draft  
**Input**: User description: 新增功能，hover到画布区组件的时候，组件展示虚线框。参考组件选中的方式区实现。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 悬停即可看到组件边界 (Priority: P1)

作为页面搭建编辑者，当我在**画布内容区**内移动指针、经过任意可编辑组件（叶节点或容器）时，我希望**立刻**看到该组件外轮廓上的**虚线框**，以便在点击选中之前就能确认「当前指针下是哪一块」，减少误点父级或空白区的情况。

**Why this priority**：这是本需求的直接价值；没有虚线框则编辑者只能靠猜测命中目标。

**Independent Test**：仅启用悬停反馈、不依赖其它新能力时，在画布上缓慢移动鼠标经过多个组件，每次指针停在某组件可视区域上时，应能稳定看到与该组件外轮廓对齐的虚线提示。

**Acceptance Scenarios**:

1. **Given** 画布上存在至少两个相邻的可编辑组件，**When** 指针依次移过每个组件的主体可视区域，**Then** 每次悬停对应组件时均出现**虚线轮廓**，且轮廓与组件外边界对齐（允许与现有缩放/网格等编辑器装饰共存，见 Assumptions）。
2. **Given** 指针位于画布区内但不在任何可编辑组件上（例如根容器留白），**When** 停留或移动，**Then** 不应错误地对不可选中的根级区域显示「子组件级」悬停框（与当前选中规则一致：根容器策略不变）。

---

### User Story 2 - 与选中态区分且互不混淆 (Priority: P2)

作为编辑者，我希望**悬停虚线**与**已选中组件的强调样式**在视觉上一眼可区分；当指针从 A 移到 B 时，A 的悬停提示应消失，B 的悬停提示应出现，而当前选中集合不因悬停而意外改变。

**Why this priority**：若悬停与选中长得一样或叠在一起无法分辨，会降低编辑信心并增加误操作。

**Independent Test**：选中一个组件后，将指针移到另一个未选中组件上，确认选中态仍保留在第一个组件上，同时第二个组件仅出现悬停虚线；指针移出所有组件后悬停虚线消失。

**Acceptance Scenarios**:

1. **Given** 组件 A 已选中且显示选中强调，**When** 指针悬停在未选中的组件 B 上，**Then** A 的选中强调仍保留，B 显示悬停虚线，二者可区分。
2. **Given** 指针悬停在组件 C 上，**When** 指针移出 C 的可视区域且未进入另一可编辑组件，**Then** 悬停虚线消失。

---

### Edge Cases

- **嵌套结构**：指针落在子组件上时，悬停框应对准**当前命中**的组件（与「从内向外解析目标组件」的交互习惯一致），不应错误地标在仅父级外框上。
- **快速移动**：指针快速划过多个组件时，悬停反馈应跟随当前指针下的组件更新，不出现长时间停留在已离开组件上的「粘滞」虚线。
- **原生控件区域**：指针在输入框、文本域、下拉等原生可编辑控件上时，不应因悬停框而阻断正常输入焦点与交互（与画布区现有指针策略对齐）。
- **缩放/滚动**：在画布缩放或滚动后，悬停轮廓仍应与组件外轮廓对齐，不出现系统性偏移。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**：在画布内容区内，当用户指针悬停在**可编辑 schema 节点**对应的主可视区域上时，系统**必须**显示与该节点外轮廓对齐的**虚线边框**。
- **FR-002**：悬停虚线在视觉样式上**必须**与**选中强调**可区分（例如线型为虚线、颜色或线宽与选中态不同）。
- **FR-003**：当指针离开当前悬停节点且未立即进入另一可编辑节点时，系统**必须**清除该悬停虚线。
- **FR-004**：悬停反馈**不得**改变当前选中集合；选中仅由既有选中交互（如点击）控制。
- **FR-005**：对**根容器**的悬停表现须与既有「根是否可选中」的产品规则一致，不得引入与规格冲突的根级误提示（见 Assumptions）。
- **FR-006**：实现方式须与「组件选中轮廓」在**定位与对齐方式**上保持一致性：即基于同一套**组件宿主边界**计算轮廓，避免悬停框与选中框系统性错位。

## Orange Editor / web_builder *(conditional)*

- **ARCH-001**：实现须遵循 `.specify/memory/constitution.md`（扩展与插槽、schema 为真源、依赖注入、store、`src/data/api`、对外导出约定）。
- **ARCH-002**：本功能仅作用于**编辑器画布区**内的 schema 渲染结果；**不**改变导出页或纯预览页的行为，除非产品明确要求（本规格默认**不**要求预览侧显示悬停框）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**：在标准测试页（不少于 5 个可编辑节点、含嵌套）中，**100%** 的节点在指针稳定悬停于其可视区域时，可在 **200ms 内**（主观可感知为「即时」）出现可辨识的虚线轮廓。
- **SC-002**：对比**同一节点**的悬停虚线与选中轮廓，在视口内随机抽测 **10** 次，**至少 9** 次两者外边界对齐误差在编辑者可接受范围内（无明显「框飘在组件外」）。
- **SC-003**：完成 **5 分钟**走查（含选中、悬停、拖移、滚动、缩放组合操作）后，**零例**因悬停层导致无法输入或无法点击原有控件的新增阻塞问题。

## Assumptions

- 「参考组件选中的方式实现」指：**对齐数据源与测量方式**应与当前选中轮廓一致（同一宿主边界），而非要求在业务页复制选中手柄等全部装饰。
- 根容器若在当前产品中**不可作为普通子组件选中**，则悬停虚线**不**应以「误提示可选中子树」的方式出现在根上；若根区域可点选以清除选中，悬停表现可与「空白区」一致、不强制虚线。
- 悬停与选中同时相关时（例如指针悬停在已选中节点上）：保留选中强调即可，**允许**叠加轻微悬停提示或仅保留选中态，但**不得**出现两种轮廓互相遮挡导致无法辨认的情况。
- 多选场景下，悬停仍只反映**指针下单一节点**，不要求为所有选中项同时显示悬停虚线。
