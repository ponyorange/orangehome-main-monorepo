# Feature Specification: 运行时预览 URL（iframe）与分享/复制链接

**Feature Branch**: `004-runtime-preview-iframe-share`  
**Created**: 2026-03-29  
**Status**: Draft  
**Input**: User description: "点击预览的时候，用iframe预览这个链接http://192.168.1.91:50055/orangehome/runtime/preview/:pageId，同时，点击分享按钮的时候，也复制这个链接。另外在预览界面，也有一个复制链接按钮，复制当前预览的链接。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 预览当前页的运行时效果 (Priority: P1)

作为编辑者，在编辑某页面并点击「预览」后，我需要在编辑器内直接看到**该页在运行时环境中的真实渲染结果**（与画布编辑视图分离），以便确认上线效果而无需手动拼接地址或打开新标签（除非我选择这样做）。

**Why this priority**: 预览是编辑闭环的核心验证步骤。

**Independent Test**: 仅实现预览区加载「当前 pageId 对应的公开预览地址」即可单独验收。

**Acceptance Scenarios**:

1. **Given** 当前上下文存在有效的页面标识（pageId），**When** 用户进入预览模式，**Then** 预览区域展示的内容来自「运行时预览服务」上该 pageId 对应的页面，而非仅画布静态渲染（若与旧行为并存，以本需求为准）。
2. **Given** 预览地址因网络或服务不可用无法加载，**When** 用户处于预览模式，**Then** 出现可理解的错误/空态提示，且可退出预览回到编辑态。

---

### User Story 2 - 分享与预览内一键复制链接 (Priority: P2)

作为编辑者，我需要**一键获得可分享的预览链接**（指向当前正在编辑页的公开预览地址）：在工具栏点击「分享」时复制到剪贴板；在预览界面内另有「复制链接」按钮，再次复制**当前预览所用**的同一链接，便于发给同事或打开系统外浏览器。

**Why this priority**: 协作与验收依赖可复制的稳定 URL。

**Independent Test**: 不依赖 Story 1 的渲染成功也可验收复制结果（例如剪贴板内容与预期 URL 一致）。

**Acceptance Scenarios**:

1. **Given** 存在有效 pageId，**When** 用户点击「分享」，**Then** 系统将该页的公开预览完整 URL 写入剪贴板，并给予成功反馈（如 Toast）。
2. **Given** 用户处于预览模式，**When** 点击预览界面上的「复制链接」，**Then** 剪贴板内容与当前预览所使用的 URL 一致（与分享按钮复制的链接规则相同）。
3. **Given** 浏览器拒绝剪贴板权限或复制失败，**When** 用户触发分享或复制链接，**Then** 用户看到明确失败提示，并可获知可选补救方式（例如手动选中链接，若产品提供展示）。

---

### User Story 3 - 环境与可配置性 (Priority: P3)

作为运维或开发者，我需要**不同部署环境**下预览基地址可配置，以便开发机、内网与生产使用各自的运行时网关，而无需改代码常量。

**Why this priority**: 硬编码 IP 仅适合单机演示，产品化需配置。

**Independent Test**: 修改配置后，预览与复制出的 URL 前缀随之变化，pageId 段仍正确。

**Acceptance Scenarios**:

1. **Given** 已配置运行时预览服务的根地址（含协议与路径前缀），**When** 生成预览 URL，**Then** 链接形态为「根地址 + 路径模板中的 pageId 占位被替换为当前页 id」。
2. **Given** pageId 缺失或非法，**When** 用户尝试预览或复制，**Then** 不生成误导性链接，并提示用户无法预览/复制的原因。

---

### Edge Cases

- 未保存的本地修改：预览展示的是**已发布/运行时可见**版本还是**当前草稿**须在实现与文档中一致；若运行时仅识别已发布数据，须在 UI 上提示「预览为已发布版本」或等价文案（假设见下）。
- 跨域与安全：嵌入的运行时页若设置禁止嵌入，预览区可能空白；产品应提供错误说明或「新窗口打开」次要入口（可选，不阻塞 MVP）。
- 剪贴板 API 在部分环境不可用（非 HTTPS、权限拒绝）：需降级提示。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 进入编辑器「预览」模式时，系统 MUST 在预览区域内嵌加载**当前页**对应的公开预览 URL（由可配置的基地址与路径规则组成，`:pageId` 替换为当前页标识）。
- **FR-002**: 用户点击「分享」时，系统 MUST 将**与预览相同的**完整预览 URL 复制到剪贴板，并给出操作反馈。
- **FR-003**: 预览界面内 MUST 提供「复制链接」控件，行为与 FR-002 一致（同一 URL 规则）。
- **FR-004**: 预览 URL 的基地址与路径模板 MUST 可通过部署配置或环境配置修改，不得将单一开发机地址写死为唯一来源（示例地址仅作开发参考，见 Assumptions）。
- **FR-005**: 在 pageId 不可用或 URL 无法构造时，系统 MUST 阻止无效复制并提示用户，不得静默失败或写入空字符串。
- **FR-006**: 复制失败时 MUST 向用户显示可理解的错误信息。

### Key Entities

- **当前页面标识（pageId）**: 与编辑器会话绑定的页面主键，用于拼预览 URL。
- **公开预览 URL**: 指向运行时预览服务、可分享给第三方的完整地址（含 pageId）。
- **预览基配置**: 协议、主机、端口（若有）、路径前缀及占位规则，决定 URL 如何拼接。

## Orange Editor / web_builder *(conditional)*

- **ARCH-001**: 实现须符合 `.specify/memory/constitution.md`（扩展/插槽、状态分层、不在核心随意挂载无关逻辑）。
- **ARCH-002**: 预览模式入口若已存在（如 `previewStore` / `Preview` 视图），应**扩展行为**而非并行造第二套预览，除非计划在规格中明确废弃旧行为。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在标准联调环境下，编辑者从点击「预览」到看到运行时页面内容（或明确错误态）的时间在 **5 秒内**可重复达成（受网络与服务 SLA 影响时在错误态中可验收）。
- **SC-002**: **100%** 的「分享」与「预览内复制链接」操作在支持剪贴板的环境下，使剪贴板中的 URL 与产品定义的预览 URL 规则一致（抽样至少 3 个不同 pageId）。
- **SC-003**: **90%** 以上受试用户能在**首次尝试**内成功完成「分享复制链接」任务（可用性小样本或走查记录）。
- **SC-004**: 更换预览基地址配置后，无需改业务代码即可使新环境预览与复制 URL 前缀正确（配置验收项）。

## Assumptions

- 用户给出的 `http://192.168.1.91:50055/orangehome/runtime/preview/:pageId` 为**开发环境示例**；规格以「可配置基地址 + `/orangehome/runtime/preview/{pageId}` 路径形态」为默认假设，实际路径以项目约定为准。
- 运行时预览页展示的数据范围（仅已发布 / 含草稿）以现有 BFF 与 runtime 契约为准；若仅支持已发布，编辑器须在预览入口附近用简短文案提示，避免用户误以为未保存内容已出现在预览中。
- 「分享」按钮在产品中已存在或将在同一迭代中一并接线；若按钮文案不同，以「复制预览链接」的语义为准。
