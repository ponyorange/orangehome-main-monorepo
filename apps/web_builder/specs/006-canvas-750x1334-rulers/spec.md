# Feature Specification: 画布标准尺寸 750×1334 与标尺一致

**Feature Branch**: `006-canvas-750x1334-rulers`  
**Created**: 2026-03-29  
**Status**: Draft  
**Input**: User description: "画布区改为现在标准的ui规范，750*1334，同时标尺也要改"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 编辑区画布采用标准幅面 (Priority: P1)

作为编辑者，我希望中间**画布工作区**所代表的页面幅面与团队约定的**标准 UI 稿尺寸**一致，即**宽 750、高 1334**（单位与编辑器内现有坐标一致），以便与设计与标注习惯对齐，减少「稿是 750 宽、编辑器却是别的尺寸」带来的换算与误解。

**Why this priority**：画布尺寸是所有摆放与标尺的基准，必须先正确。

**Independent Test**：仅打开编辑器中间画布，确认可视工作区对应的逻辑宽高为 750×1334（在默认缩放或约定基准缩放下可验收），无需改标尺即可先记录尺寸是否达标。

**Acceptance Scenarios**:

1. **Given** 用户进入页面编辑画布，**When** 查看画布所代表的设计幅面，**Then** 其**逻辑宽度为 750、逻辑高度为 1334**（与规格约定一致，允许通过界面标签或量具验收）。
2. **Given** 画布尺寸已更新，**When** 用户在画布内放置或移动组件，**Then** 坐标与边界行为仍合理（例如不出现异常裁切或无法操作的区域），且与旧版保存的页面数据兼容策略在 Assumptions 中说明。

---

### User Story 2 - 标尺与画布尺寸及缩放一致 (Priority: P2)

作为编辑者，我希望画布边缘的**横向与纵向标尺**所指示的刻度与**当前画布逻辑尺寸**一致，并在缩放画布时**同步缩放或换算正确**，以便对齐、拖拽时读数可信。

**Why this priority**：标尺是读数和精调的主要参照，必须与画布基准一致。

**Independent Test**：在画布四角及中部对照标尺读数与画布逻辑边界，切换缩放档位后再次抽样对比。

**Acceptance Scenarios**:

1. **Given** 画布逻辑尺寸为 750×1334，**When** 用户查看横向标尺，**Then** 刻度范围与画布**宽度方向**一致（起点、终点与画布可编辑宽度对应，误差在可接受展示精度内）。
2. **Given** 同上，**When** 用户查看纵向标尺，**Then** 刻度范围与画布**高度方向**一致。
3. **Given** 用户调整画布缩放，**When** 标尺与画布同时可见，**Then** 二者**相对关系保持一致**（不会出现标尺仍按旧尺寸、画布已是新尺寸的脱节）。

---

### User Story 3 - 产品内尺寸提示与文档一致 (Priority: P3)

作为编辑者或协作方，我希望界面上若展示「当前画布尺寸」类文案（如标题、提示），其**数字与 750×1334 一致**，避免口头说一套、界面写另一套。

**Why this priority**：降低沟通成本；可在核心画布与标尺稳定后收尾。

**Independent Test**：全局搜索或走查所有面向用户的画布尺寸展示，与 750×1334 核对。

**Acceptance Scenarios**:

1. **Given** 产品在某处展示画布像素尺寸，**When** 用户阅读该文案，**Then** 与 **750×1334** 一致（除非另有明确场景标签说明例外，并在 Assumptions 中列出）。

---

### Edge Cases

- **已有页面/历史数据**：旧项目以其他画布尺寸保存时，打开后布局可能需兼容（留白、裁剪或按比例提示），具体策略在 Assumptions 中约定，避免静默错位。
- **极小视口**：编辑器窗口很小时，画布通过滚动查看；标尺与滚动条的对应关系应仍可读，不出现刻度完全不可辨且无替代说明。
- **高缩放比**：刻度密度过高时，可采用抽稀显示，但**边界与关键刻度**仍应可辨或与画布对齐点一致。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**：编辑器**中心画布**所代表的设计幅面 MUST 为**逻辑宽 750、逻辑高 1334**（与当前编辑器坐标系一致，见 Assumptions）。
- **FR-002**：画布配套的**横向标尺**与**纵向标尺** MUST 与上述幅面在数值与方向上对齐，且在用户调整画布缩放时保持与画布视觉一致。
- **FR-003**：产品内面向用户展示的**画布尺寸说明**（若有）MUST 与 **750×1334** 一致，或明确标注为例外场景并列出来源。
- **FR-004**：变更后，编辑者在画布内的**基本操作**（选择、拖拽、对齐参考）MUST 仍可用，不得因尺寸切换导致核心流程不可用；若存在不兼容历史数据，MUST 有可见说明或降级策略（见 Assumptions）。
- **FR-005**：若存在「设备预览」等与画布幅面相关的展示，其**默认或推荐**行为 SHOULD 与新的标准幅面协调；若本迭代不修改预览设备列表，MUST 在 Assumptions 中写明范围边界。

### Key Entities

- **标准画布幅面**：宽 750、高 1334 的逻辑设计区域，作为编辑与标尺的基准。
- **画布缩放**：用户调整的显示比例，不改变逻辑幅面数值，但影响屏幕像素上的呈现与标尺刻度密度。
- **标尺刻度**：沿画布边缘展示的读数体系，与逻辑坐标对应。

## Orange Editor / web_builder *(conditional)*

- **ARCH-001**：实现须符合 `.specify/memory/constitution.md`（扩展/插槽、状态分层）；画布与标尺相关改动应落在既有扩展与组件边界内，避免重复多套画布常量。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**：在默认或约定的基准缩放下，**100%** 的抽样检查点确认画布逻辑尺寸为 **750×1334**（至少 3 个独立验收人或用例一致）。
- **SC-002**：标尺与画布边界在**水平与垂直两个方向**各至少 **3** 个抽样位置对齐，误差不超过产品约定的显示精度（如整像素或半像素级展示规则）。
- **SC-003**：缩放变更后，**100%** 的抽样场景（至少 **2** 个缩放级别）下，标尺与画布无「明显脱节」（走查表或录屏评审）。
- **SC-004**：界面中所有显式画布尺寸文案，走查 **100%** 与 **750×1334** 一致或已文档化例外。

## Assumptions

- **750×1334** 表示**宽 × 高**，单位为编辑器当前使用的**逻辑像素**（与现有 schema 中位置/尺寸字段一致），对应国内常见移动端 UI 稿宽度规范之一。
- **「现在标准的 UI 规范」** 在本特性中等同于采用上述固定幅面；若组织另有命名（如「2x 稿」），不改变数值要求。
- **历史数据**：若旧页面以其他画布尺寸保存，默认策略为**仍按 schema 渲染**，仅**新建或空画布**以 750×1334 为基准；若需迁移或提示，由 plan/实现阶段细化，本规格要求**不静默错误**（FR-004）。
- **预览模式 / 设备框**：若与中心画布常量分离，本规格**优先保证编辑画布 + 标尺**；预览设备列表是否同步改为 750×1334 由 plan 单列，未列入则保留现状并在 FR-005 边界中说明。
