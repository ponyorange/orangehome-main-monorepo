# Quickstart: 013-drag-drop-component-addition

## 功能验证步骤

### 测试环境准备

1. 启动开发服务器
   ```bash
   cd apps/web_builder
   rushx dev
   ```

2. 打开编辑器页面
   - 访问 http://localhost:5173
   - 创建或打开一个页面

### 测试用例 1: 拖动组件到容器

**步骤**:
1. 在画布上创建一个容器组件（如div、卡片）
2. 从左侧物料面板拖动任意组件（如按钮、文本）
3. 将鼠标移动到容器上方

**预期结果**:
- 容器显示蓝色边框高亮（2px solid #1890ff）
- 容器背景轻微变蓝（rgba(24,144,255,0.1)）
- 释放后组件被添加到容器内
- 组件左上角对齐鼠标位置

### 测试用例 2: 嵌套容器识别

**步骤**:
1. 创建嵌套容器结构：外层div → 内层div
2. 拖动组件到内层div上方

**预期结果**:
- 只有最内层容器显示高亮
- 组件被添加到内层容器而非外层

### 测试用例 3: 画布缩放测试

**步骤**:
1. 使用画布缩放控件调整缩放比例（如200%）
2. 拖动组件到任意位置

**预期结果**:
- 组件落点与实际鼠标位置偏差 <5像素
- 视觉位置和逻辑位置正确对应

### 测试用例 4: 单击添加组件

**步骤**:
1. 选中画布上的一个容器组件
2. 在物料面板中单击（非拖动）任意组件

**预期结果**:
- 新组件被添加到选中的容器内
- 位置默认为容器左上角偏移(20,20)
- 如果该位置已有组件，则向右下递增偏移

### 测试用例 5: 无选中容器时的单击添加

**步骤**:
1. 确保没有选中任何组件（点击画布空白处）
2. 在物料面板中单击组件

**预期结果**:
- 组件被添加到根画布（最外层）
- 位置为(20,20)或递增

### 边界情况测试

| 场景 | 预期行为 |
|-----|---------|
| 拖动到非容器组件（如按钮）上方 | 跳过该组件，查找其父级容器或根 |
| 鼠标超出画布边界释放 | 取消添加或限制在画布边界内 |
| 快速连续单击添加 | 每个组件位置递增偏移，不重叠 |
| 选中多个组件时单击 | 使用第一个选中的容器（如果是容器）|

### 调试信息

在浏览器控制台检查：
- `useCanvasDrop` 返回的 `dropTargetId` - 当前命中的容器ID
- 新组件的 `position.x/y` - 验证坐标转换正确性
- `selectionStore.selectedIds` - 验证选中状态

### 性能检查

- 拖拽响应应流畅，无卡顿
- 坐标计算应在 <50ms 内完成
- 连续添加10个组件应无性能下降
