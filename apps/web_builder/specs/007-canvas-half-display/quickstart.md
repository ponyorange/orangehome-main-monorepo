# Quickstart: 007 编辑区半屏展示

**前置**：`apps/web_builder` 可本地启动；已了解 006 逻辑幅面 750×1334。

## 1. 默认进入编辑器

1. 打开含 Orange 编辑器的页面，进入中心画布。
2. **视觉**：白底画布外轮廓的宽高约为「仅 006、无 007」时、**同 zoom 档** 下的约 **一半**（线性）。
3. **顶栏缩放**：默认应仍为 **100%**（或产品与改造前一致的基准文案），**不应**单独因为半屏而出现 **50%**。

## 2. 逻辑幅面与属性

1. 放置或选中一个「占满逻辑宽度」的组件（或已知宽度为 750 的用例）。
2. 在属性面板查看宽度相关展示：**仍为 750 语义**，不应变为 375。

## 3. 标尺（若开启）

1. 看横向标尺末端：应与 **整页逻辑宽度 750** 对齐，**不应**把整页标成 375。
2. 将 zoom 调到 200%：标尺与画布视觉相对关系仍可读，无「刻度与画布脱节」的明显异常。

## 4. 交互抽检

1. **拖拽**移动组件：松手后位置与预期一致，无「移一半/移双倍」的系统性偏差。
2. **缩放控制点**调整大小：宽高变化与拖拽幅度可预期。
3. **从物料区拖入**组件：落点容器正确，无系统性偏移。

## 5. 回归范围

- **不在此清单**：`Preview` 手机框尺寸（除非本迭代明确要求修改）。
