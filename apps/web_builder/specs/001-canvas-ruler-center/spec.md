# Feature Specification: 画布与横向标尺视口内水平居中

**Feature Branch**: `001-canvas-ruler-center`  
**Created**: 2026-03-27  
**Status**: Draft  
**Input**: User description: "缩小浏览器窗口宽度时，画布区和横向标尺始终在水平居中。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 窗口变窄时画布与横向标尺仍居中 (Priority: P1)

用户在可视化编辑器中编辑页面，通过拖动浏览器边缘或窗口化操作使可视区域变窄或变宽；用户期望中间的画布工作区与其上方的横向标尺作为一个整体，始终在编辑器中间留给画布的那一列里水平居中，而不是贴左或贴右导致观感失衡或难以对齐。

**Why this priority**: 直接对应原始需求，是核心体验；未满足则功能不成立。

**Independent Test**: 仅调整浏览器窗口宽度（侧栏保持默认展开状态），观察画布区与横向标尺相对中间区域的水平位置是否保持居中。

**Acceptance Scenarios**:

1. **Given** 编辑器已打开且侧栏处于常规布局，**When** 用户将浏览器窗口宽度逐步缩小，**Then** 画布区与横向标尺整体仍相对画布列水平居中，不出现仅因窗口变窄而导致的整体长期偏左或偏右。
2. **Given** 当前窗口已较窄，**When** 用户将窗口重新拉宽，**Then** 画布区与横向标尺仍保持在该列中水平居中。

---

### User Story 2 - 侧栏宽度变化后仍居中 (Priority: P2)

用户展开、收起或改变左右侧栏占用宽度后，中间画布列的可用宽度发生变化；用户期望画布区与横向标尺在新可用宽度下仍然水平居中。

**Why this priority**: 实际使用中侧栏常变动，若仅在「窗口 resize」时居中而在侧栏变化时失效，体验不一致。

**Independent Test**: 在固定浏览器窗口宽度下，仅切换侧栏展开/宽度，检查画布与横向标尺是否在中间列内重新居中。

**Acceptance Scenarios**:

1. **Given** 浏览器窗口宽度不变，**When** 用户改变左侧或右侧面板宽度（或显隐），**Then** 画布区与横向标尺在中间可用区域内仍水平居中。

### Edge Cases

- 浏览器宽度缩小到低于编辑器建议最小宽度时：居中规则仍适用；若出现滚动条或最小尺寸限制，不应在「仍可编辑」的宽度范围内无故失去居中。
- 画布内容缩放（放大/缩小视图）时：横向标尺与画布作为同一视觉/交互单元，应随该单元一起在列内居中（不与「仅标尺动、画布不动」之类分裂布局混淆）。
- 纵向标尺（若存在）：本需求不要求纵向标尺参与「水平居中」计算；仅约束画布区与**横向**标尺。
- 全屏或接近全屏的极宽窗口：居中行为与中等宽度窗口一致，不应出现仅在大宽度下才居中的情况。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 当**画布列**可用水平宽度因浏览器窗口宽度变化而改变时，系统 MUST 使**画布工作区**与**横向标尺**作为同一布局单元，在该列内水平居中。
- **FR-002**: 当**画布列**可用水平宽度因侧栏（左/右）宽度或显隐变化而改变时，系统 MUST 重新计算并保持该布局单元在该列内水平居中。
- **FR-003**: 在窗口或侧栏引起的布局更新完成后，系统 MUST 满足 FR-001 与 FR-002；若存在短暂布局更新，稳定后的状态仍须居中。
- **FR-004**: 在编辑器声明支持的窗口宽度范围内，用户 MUST 无需通过额外手动操作（例如仅为「回到中间」而拖动画布）即可恢复与窄窗/宽窗一致的居中观感。

## Orange Editor / web_builder *(conditional)*

- **ARCH-001**: 实现 MUST 遵循 `.specify/memory/constitution.md`（扩展/插槽、Schema 单一事实源、依赖注入边界、状态分层、数据 API 与对外导出约定）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在不少于 5 个明显不同的浏览器窗口宽度下（含较窄与较宽），画布与横向标尺组合相对画布列水平中心的位置偏差在可接受视觉容差内（例如结构化目测或测量下无明显系统性偏左/偏右）；侧栏布局保持默认或指定测试布局。
- **SC-002**: 在固定窗口宽度下，对侧栏进行不少于 3 种不同宽度/显隐组合操作后，上述居中关系在每次稳定后仍成立。
- **SC-003**: 在窄窗宽度下，不少于两名独立验收人员对「画布与横向标尺是否在列内水平居中」给出一致肯定结论，且无人报告必须依赖额外手动操作才能感知中线对齐。

## Assumptions

- 「画布列」指编辑器主界面中用于承载画布与横向标尺的中间栏区域，不包含左右固定侧栏；若产品将来改名，语义不变。
- 需求适用于桌面浏览器中的 Orange 可视化编辑器；未特别要求移动端专用布局。
- 未要求改变画布内容本身的文档坐标系，仅要求视口/布局层面的水平居中。
- 与现有侧栏、工具栏的叠层关系以不遮挡可读标尺刻度为前提；若存在冲突，以宪法与现有 UI 规范为准在实现阶段细化。
